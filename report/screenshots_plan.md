## Приложение Б. Скриншоты работы консольного приложения

### Как делать скриншоты
- Делай скриншот **окна консоли целиком**, чтобы были видны введённые данные и результат.
- Сохраняй в `project/report/assets/screens/`.
- Именование: `B_01_main_menu.png`, `B_02_list.png`, ... (или `.jpg`).
- В отчёте: подпись вида **«Рисунок 3.1 — Главное меню приложения»** (нумерация зависит от главы, как требует преподаватель/ГОСТ).

### Минимальный набор скриншотов (чтобы закрыть методичку)

**B-01 — Главное меню**
- Открыто приложение, видно пункты 1–11 и 0.

**B-02 — Отображение всех дел**
- Пункт `2`, вывод списка (не пустой).

**B-03 — Добавление нового дела**
- Пункт `3`, показ ввода полей и сообщение об успешном добавлении.

**B-04 — Редактирование дела по ID**
- Пункт `4`, изменение одного поля, сообщение `Дело успешно обновлено.`

**B-05 — Удаление дела**
- Пункт `5`, сообщение об удалении.

**B-06 — Поиск**
- Пункт `6`, поиск по ФИО/типу, вывод результатов.

**B-07 — Сортировка**
- Пункт `7`, выбор поля и порядка, вывод отсортированного списка.

**B-08 — Фильтрация по типу**
- Пункт `8`, ввод типа и вывод отфильтрованных данных.

**B-09 — Агрегированные показатели**
- Пункт `9`, вывод среднего/суммы/минимума/максимума.

**B-10 — Сохранение в JSON**
- Пункт `10`, ввод `out.json`, сообщение об успешном сохранении.

**B-11 — Сохранение в CSV**
- Пункт `10`, ввод `out.csv`, сообщение об успешном сохранении.

**B-12 — Загрузка из файла (JSON/CSV)**
- Пункт `11` (или подменю `1` → `3/4`), сообщение об успешной загрузке.

### Скриншоты по ошибкам ввода (желательно)

**B-13 — Некорректный пункт меню**
- Ввести число вне диапазона (например, `99`) и показать сообщение об ошибке.

**B-14 — Некорректный формат даты**
- При добавлении или редактировании ввести неверную дату и показать сообщение `Неверный формат даты...`, приложение продолжает работу.

**B-15 — Несуществующий ID при редактировании/удалении**
- Ввести ID, которого нет, показать сообщение `не найдено`.
