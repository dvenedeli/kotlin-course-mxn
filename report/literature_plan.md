## План раздела «Список использованной литературы» (≥15 источников)

Методичка требует:
- **не менее 15 источников**;
- основные источники — **не старше 5 лет** (кроме нормативных документов, справочников и т.п.);
- **ссылки по тексту** в формате `[n]` (номер в списке).

Ниже — **шаблонный набор категорий**, чтобы быстро собрать 15+ источников без «воды».

---

### 1) Kotlin / язык и стандартная библиотека (3–4 источника)
Вставлять ссылки в главу 2 (реализация), когда упоминаешь язык/основные конструкции.

- Kotlin docs: базовые типы, коллекции.
- Kotlin docs: data classes.
- Kotlin docs: работа с датами (или Java time).
- (опционально) книга/учебник по Kotlin 2020+.

### 2) JSON / сериализация (3–4 источника)
Ссылки в главу 2.3 (работа с JSON), 2.1 (модель), 3 (тестирование round-trip).

- Документация `kotlinx.serialization`.
- Документация `kotlinx.serialization-json`.
- Материал по сериализации дат (LocalDate) / custom serializers.

### 3) CSV / файловые форматы (2–3 источника)
Ссылки в главу 2.3.

- Описание CSV как формата (RFC 4180 — можно как нормативный/базовый источник).
- Рекомендации по формату и заголовку CSV.

### 4) Тестирование (3–4 источника)
Ссылки в главу 3.

- Kotlin test (`kotlin.test`) или JUnit Platform.
- Основы unit vs integration tests.
- Документация JUnit Platform Console Launcher.

### 5) Сборка Kotlin (2–3 источника)
Ссылки в главу 2 (разработка) или в приложения (как запускать).

- Kotlin compiler (`kotlinc`) — официальная документация.
- JUnit Platform User Guide (запуск тестов из командной строки).

## Как расставлять ссылки [n] по тексту (практика)

Правило: **ссылка ставится сразу после утверждения**, которое опирается на внешний источник.

Примеры:
- «Для сериализации модели в JSON используется библиотека kotlinx.serialization. [3]»
- «Формат CSV представляет собой текстовый табличный формат с разделителями. [7]»
- «Для автоматического тестирования используются unit и интеграционные тесты. [10]»
