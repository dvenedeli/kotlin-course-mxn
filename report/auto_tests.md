## Раздел отчёта: Автоматическое тестирование

### Инструменты и окружение
- **Сборка:** `kotlinc` + скрипты `run.bat` / `test.bat`; зависимости скачиваются в `lib/` по `deps.list` (`download-deps.bat`).
- **Язык:** Kotlin (JVM).
- **JDK:** 11.
- **Фреймворк тестирования:** `kotlin.test` с запуском на JUnit Platform (console launcher).

### Как запускать тесты

Из каталога `project/`:

**Windows:**
```text
test.bat
```

**Linux/macOS:**
```text
./test.sh
```

Ожидаемый результат: 17 успешных тестов, код возврата 0.

### Где лежат тесты

Каталог: `project/test/org/example/`.

Тесты разделены на:
- **Unit-тесты домена** (проверка логики без файловой системы).
- **Интеграционные тесты данных** (проверка сохранения/загрузки JSON/CSV через временные файлы).

### Перечень тестов и что они проверяют

#### 1) Unit-тесты доменной логики

Файл: `project/test/org/example/domain/NotaryCaseStoreTest.kt`

Покрытие:
- присвоение ID при добавлении;
- обновление/удаление по ID;
- поиск без учёта регистра;
- сортировка (в т.ч. по сумме);
- фильтрация по типу;
- агрегаты по `fee` (пустой список и непустой).

#### 2) Интеграционные тесты JSON

Файл: `project/test/org/example/data/JsonNotaryCaseRepositoryTest.kt`

Покрытие:
- **round-trip**: `save -> load` возвращает эквивалентный список;
- загрузка отсутствующего файла возвращает ошибку `FileNotFound`.

#### 3) Интеграционные тесты CSV

Файл: `project/test/org/example/data/CsvNotaryCaseRepositoryTest.kt`

Покрытие:
- **round-trip**: `save -> load` через CSV;
- загрузка отсутствующего файла возвращает `FileNotFound`.

#### 4) Тесты кодека CSV

Файл: `project/test/org/example/data/CsvNotaryCaseCodecTest.kt`

Покрытие:
- кодирование/декодирование списка;
- корректная обработка заголовка;
- ошибка при неверном количестве полей;
- пустой контент → пустой список.

#### 5) Проверка интеграции через слой приложения

Файл: `project/test/org/example/app/NotaryCaseAppTest.kt`

Покрытие:
- сохранение и загрузка через `NotaryCaseApp` обновляет хранилище;
- загрузка отсутствующего файла возвращает `failure`.

### Вывод по автоматическому тестированию

Автоматические тесты подтверждают корректность основной логики (CRUD/поиск/сортировка/агрегаты) и корректность операций сохранения/загрузки в JSON/CSV.
