## Приложение В. Тестовые сценарии (ручные)

Формат сценария: **ID → Цель → Предусловия → Шаги → Ожидаемый результат**.
Скриншоты результата (где требуется) сохранять в `project/report/assets/screens/` и вставлять в Приложение Б.

---

### TS-01 — Запуск приложения и автоматическая загрузка JSON
**Цель:** проверить загрузку данных из JSON при запуске.
**Предусловия:** в папке `project/` есть файл `notary_cases.json` с корректным содержимым.
**Шаги:**
1) Запустить `run.bat`.
2) В главном меню выбрать пункт `2` (отобразить все дела).
**Ожидаемый результат:**
- приложение не завершается с ошибкой;
- отображается список дел из файла;
- количество дел соответствует содержимому JSON.

### TS-02 — Отображение списка дел при пустом списке
**Цель:** проверить корректное сообщение при отсутствии записей.
**Предусловия:** загрузить пустой файл или удалить все записи через меню.
**Шаги:**
1) Выбрать пункт `2`.
**Ожидаемый результат:** выводится сообщение `Список дел пуст.`

### TS-03 — Добавление новой записи
**Цель:** проверить добавление дела и автоприсвоение ID.
**Шаги:**
1) Выбрать пункт `3`.
2) Ввести `caseNumber`, `clientName`, `caseType`.
3) Ввести дату открытия в формате `YYYY-MM-DD`.
4) Для даты закрытия нажать Enter (оставить пустой).
5) Ввести сумму гонорара.
6) Выбрать пункт `2` и убедиться, что запись появилась.
**Ожидаемый результат:**
- отображается сообщение об успешном добавлении;
- записи присваивается новый ID;
- в списке присутствует добавленная запись.

### TS-04 — Ошибка ввода: пустые обязательные поля
**Цель:** проверить защиту от пустых строк в обязательных полях.
**Шаги:**
1) Выбрать пункт `3`.
2) На запрос `Номер дела:` нажать Enter.
**Ожидаемый результат:** приложение просит повторить ввод и выводит `Поле не может быть пустым.`

### TS-05 — Ошибка ввода: дата открытия в неверном формате
**Цель:** проверить корректную обработку неверного формата даты при добавлении.
**Шаги:**
1) Выбрать пункт `3`.
2) На запрос даты открытия ввести `01-01-2024`.
3) Затем ввести корректную дату `2024-01-01`.
**Ожидаемый результат:**
- выводится `Неверный формат даты. Используйте ГГГГ-ММ-ДД.`;
- приложение не завершается;
- после корректного ввода продолжает сценарий.

### TS-06 — Редактирование записи по ID
**Цель:** проверить обновление данных существующего дела.
**Предусловия:** есть минимум 1 запись.
**Шаги:**
1) Выбрать пункт `4`.
2) Ввести существующий ID.
3) Изменить 1–2 поля (например, `clientName`, `fee`), остальные оставить пустыми.
4) Выбрать пункт `2`.
**Ожидаемый результат:**
- выводится `Дело успешно обновлено.`;
- запись с выбранным ID отображается с обновлёнными полями.

### TS-07 — Редактирование: несуществующий ID
**Цель:** проверить реакцию на ID, которого нет.
**Шаги:**
1) Выбрать пункт `4`.
2) Ввести заведомо несуществующий ID (например, `9999`).
**Ожидаемый результат:** выводится `Дело с ID 9999 не найдено.`

### TS-08 — Редактирование: неверный формат даты
**Цель:** убедиться, что неверная дата при редактировании не «роняет» приложение.
**Предусловия:** есть минимум 1 запись.
**Шаги:**
1) Выбрать пункт `4`.
2) Ввести существующий ID.
3) В поле `Дата открытия [...]` ввести `bad-date`.
4) Завершить редактирование.
**Ожидаемый результат:**
- приложение не завершается;
- выводится сообщение про неверный формат даты;
- старое значение даты сохраняется.

### TS-09 — Удаление записи
**Цель:** проверить удаление по ID.
**Предусловия:** есть минимум 1 запись.
**Шаги:**
1) Выбрать пункт `5`.
2) Ввести ID существующей записи.
3) Выбрать пункт `2`.
**Ожидаемый результат:**
- выводится сообщение об удалении;
- запись отсутствует в списке.

### TS-10 — Поиск по текстовому полю
**Цель:** проверить поиск минимум по одному текстовому полю.
**Предусловия:** есть записи с различными значениями `caseNumber/clientName/caseType`.
**Шаги:**
1) Выбрать пункт `6`.
2) Ввести часть ФИО клиента или номера дела.
**Ожидаемый результат:**
- выводится список найденных записей;
- поиск не зависит от регистра.

### TS-11 — Сортировка
**Цель:** проверить сортировку по выбранному полю.
**Предусловия:** есть минимум 3 записи с разными значениями.
**Шаги:**
1) Выбрать пункт `7`.
2) Выбрать поле сортировки (например, `5` — сумма гонорара).
3) Выбрать `y` для возрастания.
**Ожидаемый результат:** записи выведены в порядке возрастания `fee`.

### TS-12 — Фильтрация по типу
**Цель:** проверить фильтрацию по полю `caseType`.
**Предусловия:** есть записи минимум с 2 разными типами.
**Шаги:**
1) Выбрать пункт `8`.
2) Ввести часть названия типа.
**Ожидаемый результат:** выводятся только записи, тип которых содержит введённую строку.

### TS-13 — Агрегированные показатели
**Цель:** проверить расчёт avg/sum/min/max.
**Предусловия:** есть минимум 2 записи с различными `fee`.
**Шаги:**
1) Выбрать пункт `9`.
**Ожидаемый результат:**
- выводятся среднее/сумма/минимум/максимум;
- значения соответствуют `fee` у текущего списка.

### TS-14 — Сохранение в JSON
**Цель:** проверить сохранение в JSON файл.
**Шаги:**
1) Выбрать пункт `10`.
2) Ввести имя файла, например `out.json`.
**Ожидаемый результат:**
- выводится сообщение об успешном сохранении;
- файл `out.json` создаётся/перезаписывается.

### TS-15 — Сохранение в CSV
**Цель:** проверить сохранение в CSV файл.
**Шаги:**
1) Выбрать пункт `10`.
2) Ввести имя файла, например `out.csv`.
**Ожидаемый результат:**
- выводится сообщение об успешном сохранении;
- файл `out.csv` создаётся/перезаписывается;

### TS-16 — Загрузка из JSON/CSV через подменю
**Цель:** проверить загрузку через пункт `1` (подменю IO).
**Шаги:**
1) Выбрать пункт `1`.
2) Выбрать `3` (загрузка JSON) и указать файл.
3) Повторить для `4` (загрузка CSV).
**Ожидаемый результат:** выводится сообщение об успешной загрузке, список дел меняется согласно файлу.

### TS-17 — Ошибка: попытка загрузки несуществующего файла
**Цель:** проверить обработку ошибки файловой системы.
**Шаги:**
1) Выбрать пункт `11`.
2) Ввести имя файла `missing.json`.
**Ожидаемый результат:** выводится сообщение `Файл не найден: missing.json`, приложение продолжает работу.

### TS-18 — Ошибка: неверное расширение файла при сохранении/загрузке
**Цель:** проверить ограничение на `.json`/`.csv`.
**Шаги:**
1) Выбрать пункт `10`.
2) Ввести `data.txt`.
**Ожидаемый результат:** выводится `Неизвестное расширение, используйте .json или .csv`.
